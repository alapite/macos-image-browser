# Codebase Structure

**Analysis Date:** 2026-01-21

## Directory Layout

```
macos-image-browser/
├── AppState.swift                 # App state, filesystem scan, sorting, preferences, slideshow timer
├── ContentView.swift              # Root UI + multiple subviews (sidebar/viewer/settings/editor)
├── ImageBrowserApp.swift          # SwiftUI @main entry point
├── Info.plist                     # App metadata + folder access usage strings
├── Package.swift                  # SwiftPM manifest (executable target)
├── project.yml                    # XcodeGen project spec
├── build.sh                       # XcodeGen + xcodebuild + codesign pipeline
├── ImageBrowser.xcodeproj/        # Xcode project (generated/checked-in)
├── .build/                        # SwiftPM build artifacts (generated)

└── .gitignore                     # Git ignore rules
```

## Directory Purposes

**Repo root:**
- Purpose: Flat SwiftPM target layout (sources live at the top level).
- Contains: `*.swift`, `Package.swift`, build/config files.
- Key files: `ImageBrowserApp.swift`, `ContentView.swift`, `AppState.swift`.

**ImageBrowser.xcodeproj/:**
- Purpose: Xcode project (used by `xcodebuild`; may be generated by XcodeGen).
- Contains: pbxproj + scheme/workspace metadata.
- Key files: `ImageBrowser.xcodeproj/project.pbxproj`.

**.build/:**
- Purpose: SwiftPM build output.
- Contains: object files, build products.
- Committed: No (generated).

**.planning/:**
- Purpose: Planning and codebase documentation artifacts.
- Contains: `.planning/codebase/*.md`.
- Committed: Depends on project workflow.

## Key File Locations

**Entry Points:**
- `ImageBrowserApp.swift` - SwiftUI `@main` entry point.
- `Package.swift` - SwiftPM executable definition.

**Configuration:**
- `Info.plist` - App metadata and privacy usage strings.
- `project.yml` - XcodeGen spec.
- `build.sh` - Build automation.

**Core Logic:**
- `AppState.swift` - Folder selection, image enumeration, sorting, persistence.
- `ContentView.swift` - UI composition and image presentation.

**Testing:**
- Not detected (no `Tests/` directory).

**Documentation:**
- `README.md` - User/developer overview.
- `AGENTS.md` - Repo conventions and common commands.

## Naming Conventions

**Files:**
- `UpperCamelCase.swift` for Swift files (e.g., `AppState.swift`, `ContentView.swift`).
- Conventional build/config names: `Package.swift`, `Info.plist`, `project.yml`, `build.sh`.

**Directories:**
- Xcode project directory uses standard Xcode naming: `ImageBrowser.xcodeproj/`.

## Where to Add New Code

**New View / UI Component:**
- Implementation: add a new `*.swift` file at repo root (current convention), or split out existing subviews from `ContentView.swift` into dedicated files.

**New Core Logic / Service:**
- Implementation: add a new `*.swift` file at repo root and keep state mutations inside `AppState.swift` (or introduce a small service type used by `AppState`).

**Tests:**
- Add `Tests/ImageBrowserTests/` and a SwiftPM test target in `Package.swift`.

## Special Directories

**.build/:**
- Purpose: Build artifacts
- Source: Generated by SwiftPM
- Committed: No

<task type="auto">
  <name>Task 1: Refactor FileSystem abstraction for async enumeration</name>
  <files>AppState.swift</files>
  <action>
Extend the `FileSystem` abstraction (protocol or struct from Phase 1) to support async enumeration with Task cancellation:

Add to FileSystem:
- `func enumerateFiles(from folderURL: URL) async throws -> [FileEnumerationResult]`
- `FileEnumerationResult` struct: `url: URL`, `creationDate: Date?`, `error: Error?`

Implementation approach for default implementation:
- Use `FileManager.default.enumerator(at: ...)` inside an `AsyncSequence` or a loop that periodically checks `Task.checkCancellation()`.
- For each file, yield `FileEnumerationResult`.
- If cancellation is detected, throw `CancellationError`.

Keep the sync fallback for tests that need deterministic results (tests use the fake FileSystem anyway).

<task type="auto">
  <name>Task 2: Replace sync loadImages with async Task in AppState</name>
  <files>AppState.swift</files>
  <action>
Refactor `loadImages(from:)` to be async and cancellable:

- Add `@Published var isScanning: Bool = false` and `@Published var scanProgress: ScanProgress?` to AppState.
- `ScanProgress` struct: `discoveredCount: Int`, `failedCount: Int`, `totalEstimated: Int?` (optional).
- Replace `loadImages(from:)` with an async implementation that:
  1. Sets `isScanning = true`, clears previous results.
  2. Launches a detached or child `Task { }` that uses `FileSystem.asyncEnumerate(from:)`.
  3. As results arrive (via `AsyncSequence` or chunked), append to local array and update `scanProgress`.
  4. When complete, sort, assign to `images`, reset `isScanning`, trigger `@MainActor` UI updates.
- Store the `Task` reference so `selectFolder()` can call `task.cancel()` before starting a new scan.
- Use `withCheckedThrowingContinuation` or structured concurrency patterns correctly to bridge `FileManager` enumerator to async.

Constraints:
- UI must remain responsive (no main-thread blocking during enumeration).
- Existing `sortImages` behavior preserved.
- Existing `failedImages` tracking still works (populate as enumeration completes).
  </action>

<task type="auto">
  <name>Task 1: Add injectable seams for preferences + filesystem enumeration (keep behavior identical)</name>
  <files>AppState.swift, ImageBrowserApp.swift</files>
  <action>
Refactor `AppState` to accept small dependencies (with sensible defaults so the app's entry point stays simple):

Preferences seam:
- Introduce a `PreferencesStore` (protocol or small struct) responsible for save/load of `Preferences`.
- Default implementation uses `UserDefaults.standard` and the existing storage key.
- AppState uses the store instead of directly calling `UserDefaults.standard`.

Filesystem seam:
- Introduce a `FileSystem` abstraction (protocol or struct) that provides:
  - enumerating a folder for candidate URLs
  - reading file creation date (or file attributes)
- Default implementation uses `FileManager.default` with the same enumerator options and attribute keys as today.
- AppState uses the abstraction in `loadImages(from:)`.

Constraints:
- Do not change external behavior (sorting, selection, persistence key).
- Keep the seam minimal (no large "architecture" rewrite); this is just enough to make tests deterministic and enable cancellation later.
- Keep state mutations inside AppState (views should still call intent methods).
  </action>
  <verify>`swift build` succeeds and `swift run ImageBrowser` still launches</verify>
  <done>
`AppState` can be initialized with injected stores/fakes in tests while production continues to use defaults with no functional regression.
